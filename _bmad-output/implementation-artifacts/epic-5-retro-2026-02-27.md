# Retrospective - Epic 5: Feedback Fixes & UX Polish

Date: 2026-02-27
Facilitator: Bob (Scrum Master)
Participants: Vincent (Project Lead), Alice (PO), Charlie (Senior Dev), Dana (QA Engineer), Elena (Junior Dev)

## 1. Epic Overview

**Goal**: Address targeted feedback from Epic 4 to improve UX polish and data correctness before starting the Wizard dashboard work.

**Delivery Metrics:**

- **Status:** Complete (3/3 Stories Done)

## 2. What Went Well (Successes)

- **UI and Backend Decoupling:** Maintained strict semantic differences between patient-facing UI ("Added", "Filled") and canonical database statuses ("approved", boolean values).
- **Smooth Feature Delivery:** Shipped targeted chat enhancements (follow-up chips) quickly and smoothly.
- **Testing Guardrails:** Testing infrastructure and manual component testing caught regressions and edge-case state bugs early.

## 3. What Didn't Go Well (Challenges)

- **Mid-flight Requirement Changes:** Mid-flight story requirement changes during implementation caused misinterpretation and churn during code reviews, as the story files were not updated to reflect the new reality.
- **Broken Window Effect:** A lingering failing test from an earlier epic (`src/__tests__/story-1-2-database-migration.test.ts`) is creating noise in the CI.
- **State Management Fragility:** Cross-cutting concerns with Zustand state and optimistic UI wrappers broke easily across different components when dealing with parallel server actions.

## 4. Key Insights & Process Improvements

- **Documentation Sync:** We must update the markdown story files when requirements shift during implementation to avoid confusion during code review.
- **State Management Refactoring:** Refactoring the optimistic UI state management and Zustand store is needed as a technical foundation before tackling real-time subscriptions.

## 5. Preparation for Epic 6 (Creator Feedback & Core Refinements)

_Significant Discovery_: The original Epic 6 ("The Wizard's Dashboard & Scenario Control") must be deferred to Epic 7. A new Epic 6 will be created to address feedback and requests from the original creator, Kim.

2. **Kim's Account:** Implement a specific account and login screen (`/kim`) protected by a password so Kim can test the app with her true experience.
