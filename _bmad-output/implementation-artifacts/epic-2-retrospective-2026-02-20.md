## 3. What Didn't Go Well (Challenges)

- **Auth Mismatch Bug:** A critical regression in Story 2.4 where `auth.uid()` checks were mistakenly added to Server Actions. This conflicted with the simulated Persona ID architecture.
- **Fragile UX (Raw JSON):** Exposing raw JSON strings in the Glass Box edit mode in Story 2.3 & 2.4 is a UX failure and highly fragile. 
- **Requirement Misunderstanding:** The "Single List" chronolocial journal did not align with the physical journal's intent, which relies on strict categorical sorting (Appointments, Medication, Scripts).

## 4. Key Lessons & Process Improvements

### 1. The "Form Not Function" Rule
**Trigger:** User needs to edit AI-generated structured data.
**Rule:** Never expose raw JSON strings to the user. Always parse into form fields (inputs, textareas) that map back to the structured schema.

### 2. Simulated Auth Enforcement
**Trigger:** Writing Server Actions or RLS Policies.
**Rule:** All Backend logic MUST use the string Persona ID (`user_id` passed from frontend/store) and explicitly bypass `supabase.auth.getUser()`. A `// @simulated-auth-only` comment must be present.

## 5. Significant Discoveries & Scope Pivot

🚨 **SIGNIFICANT DISCOVERY:** The current chronolocial journal view does not match the product vision. The application must reflect a "Section-Based Health Record" (Navigation tabs for Home, Appointment, My Medication, Scripts & Referrals) mirroring the physical journal.

This fundamentally obsoletes the original plan for Epic 3 as "Wizard Dashboard", since the core data taxonomy and navigation needs an overhaul first. 

### **Resulting Action:**
Pivot Epic 3 to **"Structured Health Records & Navigation"**. The Wizard Dashboard will be delayed to Epic 4.
